แนวคิดหลัก (Core Concept)

Media Queries คืออะไร?

คือคำสั่ง CSS ที่ช่วยให้เรากำหนดสไตล์ที่แตกต่างกันได้ตามขนาดของหน้าจอ (Viewport Width)

@media (เงื่อนไข) {
  /* CSS ทำงานเมื่อเงื่อนไขเป็นจริง */
}
min-width

ตั้งแต่ขนาดนี้ ขึ้นไป (ใช้กับ Mobile First)

max-width

ห้ามเกินขนาดนี้ / ตั้งแต่ขนาดนี้ ลงมา (ใช้กับ Desktop First)

ข้อแนะนำ: กำหนด Breakpoints อย่างไรไม่ให้สับสน

แนะนำ กลยุทธ์ 1: Mobile First

เริ่มเขียน CSS พื้นฐานสำหรับมือถือก่อน แล้วค่อยใช้ min-width เพื่อ "เพิ่ม" สไตล์เมื่อจอใหญ่ขึ้น

// พื้นฐาน (มือถือ) อยู่บนสุด

.container { width: 100%; }


@media (min-width: 576px) { ... } // Tablet S

@media (min-width: 768px) { ... } // Tablet M

@media (min-width: 992px) { ... } // Laptop

ข้อดี: โค้ดเป็นระเบียบ เรียงจากเลขน้อยไปมาก ไม่สับสน ทับซ้อนยาก

ห้ามทำ สิ่งที่ทำให้สับสน (Bad Practices)

  • 1. ใช้ min-width ผสม max-width มั่ว

    การผสมกันโดยไม่มีแผนจะเกิด "ช่องว่าง" (เช่น 768px ถึง 800px ไม่มีกฎไหนรองรับ) หรือสไตล์ตีกันเอง

  • 2. เรียงลำดับผิด

    ใน CSS กฎที่มาทีหลังจะทับกฎก่อนหน้าเสมอ ถ้าใช้ min-width ต้องเรียง น้อย -> มาก เสมอ

Tip: ปัญหาจุดทศนิยม
หลีกเลี่ยงการใช้ max-width: 767px คู่กับ min-width: 768px เพราะหน้าจอขนาด 767.5px อาจไม่เข้าเงื่อนไขทั้งคู่
ทางแก้: เลือกใช้ระบบ Mobile First (min-width ล้วน) จะแก้ปัญหานี้ได้ 100%
